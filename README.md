# When Was the Last Time I — API

REST API for **When Was the Last Time I**, an app for tracking reminders inside shared groups. Users sign up, verify email, and manage **reminders** (with completion history and steps) under **reminder groups** with role-based access (**owner**, **admin**, **member**, **invited**).

## Tech stack

| Layer | Technology |
|--------|------------|
| Runtime | Node.js (build targets **Node 22** on Vercel) |
| Language | TypeScript (**strict**, ESM, `.ts` imports) |
| HTTP | Express 5 |
| Database | MongoDB via **Mongoose** |
| Auth | **JWT** (`jsonwebtoken`), password hashing with **bcrypt** |
| Email | **Nodemailer** (registration, password reset, etc.) |
| CORS | **cors** |
| Deployment | **Vercel** serverless (`vercel.json` + esbuild bundle) |

## Prerequisites

- **Node.js** 22 or newer (recommended; matches the Vercel bundle target)
- A **MongoDB** instance (connection string)
- For email flows: SMTP-compatible credentials (see environment variables)

## Installation

```bash
git clone https://github.com/carloswr0/when-was-the-last-time-i-api.git
cd when-was-the-last-time-i-api
npm install
```

Copy the environment template and fill in values:

```bash
cp .env.sample .env
```

### Environment variables

Configure these in `.env` (or in the Vercel project settings for production):

| Variable | Purpose |
|----------|---------|
| `MONGO_DB_CONNECTION_STRING` | MongoDB connection URI |
| `PORT` | HTTP port for local server (defaults to **3000** in code if unset) |
| `JWT_SECRET_KEY` | Secret for signing JWT access tokens |
| `MAIL_EMAIL` | Sender / SMTP user for Nodemailer |
| `MAIL_PASSWORD` | SMTP password or app password |
| `URL_FRONTEND` | Frontend base URL (e.g. links in emails) |
| `URL_BACKEND` | Backend base URL where this API is exposed |

## Local development

Start the API with file watching (restarts on changes):

```bash
npm run dev
```

The server listens on `PORT` from `.env`, or **3000** if `PORT` is not set.

**Plain start** (no watch):

```bash
npm start
```

**Vercel locally** (builds the serverless bundle then runs `vercel dev`):

```bash
npm run vercel-dev
```

Requires the [Vercel CLI](https://vercel.com/docs/cli) and a linked project.

### API route overview

- `GET /api/health` — API health check
- `GET /api/health/db` — MongoDB connectivity check
- `POST /api/auth/*` — register, login, verify email, password reset
- `GET|POST|PUT|DELETE /api/groups/*` — groups, invitations, membership
- `GET|POST|PATCH|DELETE /api/groups/:group_id/reminder/*` — reminders scoped to a group
- `GET /api/user/:user_id/get-all-reminders` — aggregated reminders (authenticated)
- `GET /api/test` — authenticated smoke test

All routes under `/api/groups`, `/api/user`, and `/api/test` expect a valid JWT (see `auth.middleware.ts`).

## Deployment (Vercel)

1. Connect this repo to [Vercel](https://vercel.com) and set **Root Directory** if the repo is monorepo-style; for this repo, root is the project root.
2. Set the same environment variables as in `.env` in the Vercel project **Settings → Environment Variables**.
3. Deploy: Vercel runs `npm run build`, which bundles `vercel-entry.ts` with **esbuild** to `api/index.js` and treats it as the serverless entry. `vercel.json` rewrites incoming requests to `/api`.

Ensure `MONGO_DB_CONNECTION_STRING`, `JWT_SECRET_KEY`, and mail/URL variables are set for the deployment environment (Production / Preview).

## Project structure

```
when-was-the-last-time-i-api/
├── src/
│   ├── app.ts                 # Express app: middleware, route mounting
│   ├── index.ts               # Local server entry (listens unless VERCEL is set)
│   ├── config/                # env, MongoDB, mailer
│   ├── constants/             # shared constants (e.g. error codes)
│   ├── controllers/          # HTTP handlers (auth, groups, reminders, user, health)
│   ├── helpers/              # responses, errors, auth helpers
│   ├── middlewares/          # JWT auth, group/reminder checks, role checks
│   ├── models/               # Mongoose schemas (User, Reminder, groups, history, steps)
│   ├── repositories/         # Data access layer
│   ├── routers/              # Route definitions (nested reminders under groups)
│   ├── services/             # Business logic (auth, reminders)
│   └── types/                # Shared TypeScript types (e.g. API responses)
├── vercel-entry.ts            # Serverless export for Vercel (`export default app`)
├── vercel.json                # Vercel build + rewrite rules
├── package.json
├── tsconfig.json
├── .env.sample                # Environment variable template
└── api/index.js               # Generated by `npm run build` (gitignored)
```

**Flow:** Routers call controllers; controllers use services and repositories; repositories use Mongoose models. Middleware enforces authentication and group/reminder membership and roles before mutating group-scoped resources.
